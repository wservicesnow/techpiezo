/tmp on RAM

  • By Editorial Team
  • On Jun 11, 2019
  • In Insights

The purpose of /tmp directory on GNU/Linux has been to provide programs a directory for temporary files. Files in /tmp are deleted after a system reboot.

Many GNU/Linux based distributions have mounted /tmp as RAM based tmpfs (temporary file system). It not only removes disk I/O but also increases performance. However, it has certain aspects to clearly understood.

What /tmp on RAM disk provides:

A. Performance

Since /tmp will be mounted on ram disk as tmpfs, Read and Write will be significantly faster as RAM’s Read and Write speeds are faster then that of Hard Drive or Solid State Drive. Hence programs accessing data on /tmp will be faster.

B. Disk Optimization

Temporary files been placed on tmpfs, frees disk I/O which allow disk activity to go down and allocate resources to other disk I/O operations. Specially fewer Read and Write to Flash based Storage reduces the requirement to TRIM NAND based Disk.
Also fewer Disk wake-ups.

C. Data Security

As temporary data is not written to disk, the chances of data leak from Hard Disk/SSD are none.

What are limitations of /tmp on RAM disk:

A. Pseudo Data Security

Data can however still leak, while system is live and program’s data resides in tmpfs or other side channels depending upon security measures taken by the program.

But data will not leak after device is powered down, with exception for cold boot method for data recovery.

B. RAM Limit

tmpfs size is limited to amount of RAM provided in the system, hence /tmp will not be able to grow beyond limit. If program hogs on /tmp directory for huge amounts of data then system will resort back to swap space or slow down the whole system, leaving you with no RAM for other processes.

Configure /tmp on tmpfs

On RHEL 7 / CentOS 7 / Fedora:

To Check

systemctl is-enabled tmp.mount

To Enable

systemctl enable tmp.mount

To Disable

systemctl disable tmp.mount

On Debian Stretch :

To Check

systemctl is-enabled tmp.mount

To Enable

cp /usr/share/systemd/tmp.mount /etc/systemd/system/

systemctl enable tmp.mount

To Disable

systemctl disable tmp.mount

Manually On GNU/Linux :

Add to /etc/fstab

tmpfs /tmp tmpfs rw,nosuid,noatime,nodev,size=4G,mode=1777 0 0

*size=4G depending on ram allocation.

Manually resize on fly :

mount -o remount,size=4G,noatime /tmp
